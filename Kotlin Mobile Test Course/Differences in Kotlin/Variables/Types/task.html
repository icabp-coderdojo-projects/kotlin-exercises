Variable types

When working with Javascript or Python, you may have heard of the term 'type', for example:

"Example" is a String
16 is an Integer
[2, 3, 4] is an Array

and so on, for all 'types' of data we want to store

Types are important when programming because our programs need to know what to do with data. We can't add an integer the same way we add a string
You might have seen this if you ever tried adding 1 + 1, or "1" + "1" in Javascript or Python
In the first case, we get 2, but in the second case, we get "11"

This is because strings could be "Hello", and "world", and when we add them we want to get "Hello world"

It looks confusing because we see numbers inside our string, but they are also a String type, and get added the same way

This behaviour is because of types, which each variable is secretly given when you create it

Soft typed vs. Hard typed

Some languages are called soft typed languages.
These languages will handle the types of variables for you
 - Javascript and Python are soft typed

Other languages are called hard typed languages, which do not handle variable types.
 - Kotlin is hard typed

For example: 

  function add(x, y) {
    let result = x;
    for(i=0; i < y; i++) {
      result++;
    }
    return result;
  }
works in Javascript for any two integers, the same function in Kotlin looks like (also in the Task.kt file): 

  fun add(x: Int, y: Int): Int {
    var result = x
    for(i in 0 until y) {
        result++
    }
    return result
  }
This is very bad code (and you should never write it), but works as an example

Try giving our Kotlin function a String for x, y, or both, and see what happens

You can also try the Javascript function in the Chrome/Firefox console if you want, or just look at the hint below: 

    >>add(1, 2)
    - 3
    >>add("Hello", "world")
    - "Hello"
    >>add(4, "world")
    - 4
As you can see, in Kotlin our function will either work or give an error, but in Javascript it will always try run, giving us the wrong output without warning because of it handling types for us

This is one reason some people prefer hard typed over soft typed, but both types of language have many benefits and downsides to consider
